# Contributing to Classic Snake Game

Thanks for your interest in contributing! We welcome improvements, bug fixes, documentation updates, and ideas for features.

How to contribute

1. Fork the repository to your GitHub account.
2. Create a feature branch (`git checkout -b feature/my-change`).
3. Make your changes with clear commits and descriptive messages.
4. Run and test your changes locally.
5. Push your branch to your fork and open a Pull Request against the upstream `main` branch.

Guidelines

- Keep changes small and focused. One logical change per PR makes review easier.
- Follow the existing code style. This project uses TypeScript with modest formatting conventions.
- Include tests or manual test instructions for behavioral changes when applicable.
- Update documentation (`README.md`, inline comments) for public APIs or UX changes.

Reporting issues

- Use the GitHub Issues page to report bugs or request features.
- Provide a clear summary, steps to reproduce, expected vs actual behavior, and environment details (browser, OS).

Code of conduct

By participating you agree to follow the project `CODE_OF_CONDUCT.md`.

Thank you for helping improve this project!
